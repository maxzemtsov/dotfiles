# UX Psychology: 106 Cognitive Biases & Principles

Source: [Growth.Design Psychology](https://growth.design/psychology) by Dan Benoni & Louis-Xavier Lavallée.

Every user interaction involves four cognitive stages:
1. **Filter information** — users ignore most of what they see
2. **Seek meaning** — users fill gaps with assumptions and stories
3. **Act within time** — users take shortcuts and jump to conclusions
4. **Store in memory** — brains prefer some elements over others

Apply these principles at every stage of product design. When making UX decisions, reference the relevant biases from the catalog below. For detailed principle descriptions and actionable tips, read the reference files:

- **Information biases** (29 principles) → [references/information.md](references/information.md)
- **Meaning biases** (32 principles) → [references/meaning.md](references/meaning.md)
- **Time biases** (28 principles) → [references/time.md](references/time.md)
- **Memory biases** (17 principles) → [references/memory.md](references/memory.md)

## How to Use This Skill

When designing or reviewing any product feature:

1. **Identify the UX stage** — Is this about presenting information? Helping users understand? Driving action? Creating lasting impressions?
2. **Select relevant principles** — Use the quick-reference table below to find applicable biases
3. **Apply concretely** — Turn each principle into a specific design decision with rationale
4. **Check for conflicts** — Some biases pull in opposite directions; make deliberate trade-offs
5. **Prioritize ethically** — Use biases to help users achieve their goals, not to manipulate

## Ethical Guidelines

These principles are tools, not weapons. Apply them to:
- Help users achieve their goals faster
- Reduce friction and cognitive effort
- Create genuinely delightful experiences
- Build trust through transparency

Never use them to:
- Trick users into actions they wouldn't otherwise take
- Create artificial urgency or false scarcity
- Hide important information behind dark patterns
- Exploit emotions for short-term gains at user expense

When reviewing designs, flag any dark pattern usage and suggest ethical alternatives.
